# 客户端原生 PyQt 改造工程进度追踪

## 使用说明

- 本文件用于跟踪“客户端页面全部改造成原生 PyQt”的整体进度。
- 每个任务默认状态分为：`未开始`、`进行中`、`已完成`、`阻塞`。
- 建议每次开发结束后更新：状态、完成日期、负责人、备注。
- 建议以本文件作为每周例会或阶段汇报的唯一进度来源。

## 项目目标

- 将客户端从“原生小窗 + 外部浏览器跳转”改造成“原生主工作台 + 原生业务页面 + 浏览器兜底”。
- 优先完成医生/护士高频模块原生化，再逐步覆盖管理员后台模块。
- 保留现有登录、悬浮窗、托盘、待办、整改、详情等已完成原生能力，并纳入统一主工作台。

## 当前现状

- 已原生化：登录框、原生主工作台、悬浮提醒窗、我的待办、整改确认、待办详情、病例讨论、业务学习、病历检查、患者回访、任务分配、用户管理、数据汇总、患者导入。
- 未原生化：无主业务阻塞项，浏览器入口仅作为备用能力保留。
- 当前客户端主入口、任务处理链、托盘菜单与悬浮窗菜单均已切换为原生优先。
